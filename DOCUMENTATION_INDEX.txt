Common Commands:

Development:
  npm run dev                    # Start both frontend + backend
  npm run dev:frontend          # Start frontend only (port 5173)
  npm run dev:backend           # Start backend only (port 3001)
  npx prisma studio             # Open database UI (port 5555)

Database:
  npx prisma generate           # Generate Prisma client
  npx prisma migrate dev        # Run migrations
  npx prisma db push            # Push schema to database

Production:
  npm run build                 # Build frontend
  npm start (in backend/)       # Start Node.js server

Caching:
  GET /api/cache/stats          # View cache contents
  POST /api/cache/clear         # Clear cache patterns

Common API Endpoints:
  POST /api/auth/login          # Login
  GET  /api/maintenance/records # Get records (with caching)
  GET  /api/cache/stats         # View cache status
  GET  /api/health/detailed     # System diagnostics


SECURITY CHECKLIST FOR REVIEWERS
================================================================================

☑ Authentication:
  □ JWT tokens verified on protected routes
  □ httpOnly cookies prevent XSS
  □ sameSite=strict prevents CSRF
  □ 24-hour token expiration enforced

☑ Authorization:
  □ Role-based access control (ADMIN/User)
  □ Branch filtering prevents cross-branch access
  □ Cost code filtering with fuzzy matching
  □ Empty permission lists explicitly deny all

☑ Input Validation:
  □ All endpoints validate input with Joi schemas
  □ Date parameters validated (YYYY-MM-DD format)
  □ Role values whitelisted (only ADMIN/User)
  □ Branch values whitelisted (only PMB/PTA/QTN/CPT/ALL)
  □ String lengths limited
  □ Array contents validated

☑ Database Security:
  □ Passwords hashed with bcrypt (10 rounds)
  □ Parameterized queries prevent SQL injection
  □ MSSQL encryption enabled
  □ No credentials hardcoded (use environment variables)

☑ API Security:
  □ Rate limiting on login (10 attempts/15 min)
  □ Rate limiting on data (30 requests/min)
  □ Rate limiting on admin (20 requests/5 min)
  □ Payload size limited to 10KB
  □ Helmet security headers configured
  □ CSP headers configured

☑ Infrastructure:
  □ HTTPS enforced (secure cookies)
  □ .env file not committed to git
  □ .env.example template provided
  □ No secrets in code or comments

TROUBLESHOOTING GUIDE
================================================================================

Issue: "Cannot connect to database"
Symptoms: ECONNREFUSED, Connection timeout
Solution: Check server IP, port, credentials
See: RELEASE_NOTES_COMPREHENSIVE.txt → "Troubleshooting"

Issue: "Token expired"
Symptoms: 401 Unauthorized
Solution: User must login again (future: implement refresh tokens)

Issue: "Slow response times"
Symptoms: Request takes 4+ seconds
Solution: Check cache stats, verify data hasn't changed
Command: GET /api/cache/stats

Issue: "Too many connections"
Symptoms: "Connection pool exhausted" errors
Solution: Increase pool.max in dbManager.js
Current: max: 20 (handles ~50 users)

Issue: "High memory usage"
Symptoms: Node process using >500MB
Solution: Enable pagination, check cache TTL
Command: GET /api/cache/stats
